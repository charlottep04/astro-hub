La transpilation TypeScript vers JavaScript

La transpilation est le processus de conversion du code TypeScript en code JavaScript. TypeScript est un sur-ensemble de JavaScript qui ajoute des fonctionnalités supplémentaires comme le typage statique. Le compilateur TypeScript (tsc) prend le fichier .ts et génère un fichier .js équivalent en supprimant les annotations de type et en conservant la logique du code.

Comparaison des fichiers

TypeScript (hello.ts) :

  • Contient des annotations de type (: string).
  • Utilise des fonctionnalités de typage statique pour vérifier les types à la compilation.
  • Utilise const pour déclarer une constante.

JavaScript (hello.js) :

  • Ne contient pas d'annotations de type.
  • Utilise var pour déclarer une variable, ce qui a une portée différente et peut entraîner des comportements inattendus par rapport à const.

Utilité du typage fort en TypeScript

Le typage fort en TypeScript offre plusieurs avantages par rapport à JavaScript :

Détection des erreurs à la compilation :

  • Les erreurs de type sont détectées lors de la compilation, ce qui permet de les corriger avant l'exécution du code.

Maintenance et lisibilité du code :

  • Le typage explicite rend le code plus facile à comprendre et à maintenir car les types des variables et des fonctions sont clairement définis.

Réduction des bugs :

  • Le typage statique aide à prévenir les erreurs courantes liées aux types comme l'utilisation de méthodes sur des types incorrects.
